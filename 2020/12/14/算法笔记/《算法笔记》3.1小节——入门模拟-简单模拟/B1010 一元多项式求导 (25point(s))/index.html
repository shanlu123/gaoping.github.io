PAT乙 1010 一元多项式求导 (25point(s))

发表于 2020-12-14 分类于 OJ--《算法笔记》 阅读次数： 阅读次数： Valine： 本文字数： 3.3k 阅读时长 ≈ 3 分钟

题目

18分代码

思路

  1. 开一个数组 arr，arr[ key ] = value 表示指数是key的项的系数是value
  2. 通过 while….EOF 方式输入每一组系数和指数。同时定义一个 count 变量用来记录一共输入了几组数，即多项式中一共有多少项。因为是按照指数递减的顺序输入，所以循环结束的条件就是输入的指数小于等于0时。
  3. 输入完毕，开始根据count值遍历 arr数组，数组中的每一个元素可以表示多项式其中一项的指数和系数，因此，遍历该数组，先求出求导后的系数，再将求导后该项的指数减一。
  4. 修改 arr数组的系数和指数时，0次项（常数项）直接处理为0，同时记录0次项的个数，最后打印的时候需要做特殊判断。
  5. 修改之后的arr数组就是每一项求导后的指数和系数，倒着遍历该数组，打印出求导后的指数和系数即可。
  6. 在打印最后求导的结果时，要注意两个点，①所有常数项求导后的指数和系数都是0，不打印。②先前已经记录了0此项的个数，如果0次项的个数等于输入的总的个数，就输入 0 0，程序结束。

问题

  • 修改数组遍历时，将指数的范围限定在 0-count，但是可能输入4个数，指数的范围超过了4，

22分代码

修改

  1. 由于count变量设置不正确，所以这里删除了count 计数，而是直接遍历到 1000，因为指数从0到1000都有可能。
  2. 为了判断是否求导后都是0，设置了一个 flag标志，用来做求导结果中系数不为0的项的计数。如果 flag==0，说明没有求导后系数都是0，直接打印 0 0 即可。
  3. 另外注意这里 max的上限设置高一些，arr [max]，将 max 理解为指数时范围是0-1000，但是理解为数组的长度，max 至少是1001，所以max 可以再设置高一些，否则打印出奇怪的数字。
  4. 这里应该明白一个点，在录入输入的每一项后，如果输入的常数项不止一项，最后会被覆盖，所以待求导的数组中，常数项只会有一项。
