多表(连接)查询

发表于 2020-09-18 更新于 2020-09-19 分类于 数据库 阅读次数： 阅读次数： Valine： 本文字数： 3.8k 阅读时长 ≈ 3 分钟

连接查询

  • 又称多表查询，当查询的字段来自于多个表时，就需要用到连接查询
  • 分类：
    • 按年代分类：
      • sql92 标准：仅支持内连接
      • sql99 标准：支持内连接 + 外连接 + 交叉连接
    • 按功能分类：
      • 内连接：等值连接、非等值连接、自连接
      • 外连接：左外连接、右外连接、全外连接
      • 交叉连接
  • 笛卡尔乘积现象
    • 表1有m行，表2有n行，连接后结果为 m*n 行
    • 原因：没有添加有效的连接条件
    • 解决：可以在Where 子句中加入有效的连接条件
  • sql92和sql99
    • sql99 支持的更多
    • sql99 实现连接条件和筛选条件的分离，可读性较高

等值连接（sql 92)

  • 等值连接的结果为多个表的交集部分
  • n个表连接，至少需要 n-1 个连接条件
  • 多表连接，对顺序没有要求
  • 一般需要为每个表起别名
  • 可以搭配排序，分组，筛选等子句使用

等值连接(sql 99)

  • 通过 join 关键字实现连接
  • 通过 ON 子句创建连接
    • 自然连接中是以具有相同名字的列为连接条件的
    • 可以使用ON 子句指定额外的连接条件
  • 分类：
    • 内连接：[inner] join on
    • 左外连接：left [outer] join on
    • 右外连接：right [outer] join on

外连接（仅sql99支持)

  • 应用场景：用于查询一个表中有，另一个表中没有的记录

  • 使用：

    • 外连接的查询结果是主表中的所有记录，如果从表中有和它匹配的，则显示匹配的值；如果从表中没有和它匹配的，则显示 null；
    • 外连接的查询结果=内连接查询结果+主表中有而从表中没有的记录

  • 左外连接：left join 左边的是主表

  • 右外连接：right join 右边的是主表

  • 全外连接：内连接的结果+表1中有但表2中无+表2中有但表1中无
