子查询

发表于 2020-09-19 分类于 数据库 阅读次数： Valine： 本文字数： 3.9k 阅读时长 ≈ 4 分钟

概念

  • 主查询：外部的查询语句，称为主查询或外查询
  • 子查询：一条查询语句中又嵌套了另一条完整的select语句，其中被嵌套的select语句，称为子查询或内查询
    • 子查询的执行优先于主查询执行，主查询的条件用到了子查询的结果
    • 子查询放在小括号内
    • 子查询可以放在from后面，select后面，where后面，having后面，但一般放在条件的右侧

子查询分类：

  • 按结果集的行列数不同：
    • 标量子查询：结果集只有一行一列，一般搭配单行操作符使用(> < >= <= = <>)
    • 列子查询：结果集只有一列多行，一般搭配多行操作符使用(in，any/some，all)
    • 行子查询：结果集只有一行多列
    • 表子查询：结果集一般为多行多列
  • 按子查询出现的位置分：
    • select 后：仅仅支持标量子查询
    • from 后：支持表子查询
    • where或having 后：标量子查询+列子查询+行子查询
    • exitsts后（相关子查询）：表子查询

多表(连接)查询

发表于 2020-09-18 更新于 2020-09-19 分类于 数据库 阅读次数： Valine： 本文字数： 3.8k 阅读时长 ≈ 3 分钟

连接查询

  • 又称多表查询，当查询的字段来自于多个表时，就需要用到连接查询
  • 分类：
    • 按年代分类：
      • sql92 标准：仅支持内连接
      • sql99 标准：支持内连接 + 外连接 + 交叉连接
    • 按功能分类：
      • 内连接：等值连接、非等值连接、自连接
      • 外连接：左外连接、右外连接、全外连接
      • 交叉连接
  • 笛卡尔乘积现象
    • 表1有m行，表2有n行，连接后结果为 m*n 行
    • 原因：没有添加有效的连接条件
    • 解决：可以在Where 子句中加入有效的连接条件
  • sql92和sql99
    • sql99 支持的更多
    • sql99 实现连接条件和筛选条件的分离，可读性较高

等值连接（sql 92)

  • 等值连接的结果为多个表的交集部分
  • n个表连接，至少需要 n-1 个连接条件
  • 多表连接，对顺序没有要求
  • 一般需要为每个表起别名
  • 可以搭配排序，分组，筛选等子句使用
